2. Sub-Agent Relay Protocol

When a coordinator processes a sub-agent event (register, done, blocker, progress), it relays a sub_agent_status message to the queen. Supports arbitrary nesting depth.

sequenceDiagram
    participant SA as sub-scout-1
    participant CO as Coordinator
    participant QI as Queen Inbox
    participant QS as Queen State

    SA->>CO: register (via QUEEN_INBOX)
    Note over CO: handleQueenMessage()
updateAgentStatus("sub-scout-1", "running")
    CO->>QI: sub_agent_status {from: "coord", subAgent: {name: "sub-scout-1", status: "running"}}
    QI->>QS: upsertSubAgent("coord", {name: "sub-scout-1", status: "running"})
    Note over QS: agents.set("sub-scout-1",
{spawnedBy: "coord", status: "running"})

    SA->>CO: done {summary: "mapped files"}
    Note over CO: updateAgentStatus → relaySubAgentStatus
    CO->>QI: sub_agent_status {subAgent: {name: "sub-scout-1", status: "done"}}
    QI->>QS: upsertSubAgent → update status to "done"
    Note over QS: Dashboard updates ✓
checkAllDone skips sub-agents ✓

4. Dynamic Agent Addition (swarm_add)

NEW TOOL Add agents to a running swarm mid-flight. Creates channels, spawns processes, wires lifecycle.

flowchart TD
    START([swarm_add called]) --> CHECK{Active swarm?}
    CHECK -- No --> ERR1[❌ No active swarm]
    CHECK -- Yes --> NAMES{Name collisions?}
    NAMES -- Yes --> ERR2[❌ Name collision]
    NAMES -- No --> LOOP

    subgraph LOOP ["For each agent"]
        direction TB
        CH[1. group.addChannel
create inbox socket] --> CONN[2. connectToChannel
queen monitors inbox]
        CONN --> STATE[3. Register in
SwarmState.agents]
        STATE --> SPAWN[4. spawnAgent
with full roster]
        SPAWN --> WIRE[5. wireAgentProcess
crash detection, timeout]
        WIRE --> ANNOUNCE[6. Broadcast agent_added
on general channel]
    end

    LOOP --> DASH[Update dashboard]
    DASH --> DONE([✅ Return summary])
